## üß† **1. What are Wrapper Classes in Java?**

In Java, **wrapper classes** are **object representations of primitive data types**.
Java is an **object-oriented language**, but primitive types like `int`, `char`, `boolean` are **not objects**.
So, Java provides **wrapper classes** in the `java.lang` package to ‚Äúwrap‚Äù primitive values inside objects.

| Primitive Type | Wrapper Class |
| -------------- | ------------- |
| `byte`         | `Byte`        |
| `short`        | `Short`       |
| `int`          | `Integer`     |
| `long`         | `Long`        |
| `float`        | `Float`       |
| `double`       | `Double`      |
| `char`         | `Character`   |
| `boolean`      | `Boolean`     |

---

## üß∞ **2. Why Wrapper Classes Are Needed**

Wrapper classes are useful when:

* ‚úÖ You want to use primitive values in **collections** like `ArrayList`, `HashMap` (which only store objects).
* ‚úÖ You need to **convert primitives to strings** and vice versa.
* ‚úÖ You want to use **utility methods** (e.g., `Integer.parseInt()` or `Character.isDigit()`).
* ‚úÖ You need to **pass primitives as objects** to methods.

### üîÑ **4. Autoboxing & Unboxing ‚Üí Automatic conversion**

```java
import java.util.ArrayList;

public class AutoBoxingDemo {
    public static void main(String[] args) {
        // Autoboxing: primitive ‚Üí object automatically
        Integer num = 10;   // Compiler converts to Integer.valueOf(10)
        ArrayList<Integer> list = new ArrayList<>();
        list.add(20);       // Autoboxed

        // Unboxing: object ‚Üí primitive automatically
        int val = num;      // Compiler converts to num.intValue()
        int sum = val + list.get(0);  // list.get(0) is unboxed

        System.out.println("Value of num: " + num);
        System.out.println("List element: " + list.get(0));
        System.out.println("Sum: " + sum);

        // ‚ö†Ô∏è NullPointerException if unboxing null
        Integer n = null;
        // int x = n; // This would throw NPE!
    }
}
```

‚úÖ **Key Concepts:**

* Introduced in Java 5.
* Happens automatically during assignments, arithmetic, and method calls.
* ‚ö†Ô∏è Unboxing null causes `NullPointerException`.

## üß† **1. What is Autoboxing?**

üëâ **Autoboxing** is the **automatic conversion** by the Java compiler of a **primitive type ‚Üí its corresponding wrapper class object**.
This was introduced in **Java 5** to make code cleaner and more readable.

### üî∏ Example:

```java
int num = 10;
Integer obj = num;   // ‚úÖ Autoboxing happens here
```

üëâ Internally, the compiler does something like:

```java
Integer obj = Integer.valueOf(num);
```

### ‚úÖ Why Autoboxing is Needed:

* Java Collections (e.g., `ArrayList`) can **store only objects**, not primitive values.
* Earlier, we had to manually convert primitives to objects using methods like `Integer.valueOf()`.
* Autoboxing removes this **manual wrapping**.

---

## üß† **2. What is Auto-unboxing?**

üëâ **Auto-unboxing** is the **reverse process** ‚Äî automatic conversion of a **wrapper object ‚Üí its corresponding primitive type**.

### üî∏ Example:

```java
Integer obj = 20;
int num = obj;   // ‚úÖ Auto-unboxing happens here
```

üëâ Internally, the compiler does:

```java
int num = obj.intValue();
```

### ‚úÖ Why Auto-unboxing is Needed:

* When you perform arithmetic operations or use wrapper objects in places where primitives are expected, Java automatically converts them back.

---

## üß™ **3. Example Program: Autoboxing and Auto-unboxing**

```java
public class AutoBoxingUnboxingExample {
    public static void main(String[] args) {
        // üëá Primitive type
        int a = 100;

        // ‚úÖ Autoboxing: primitive ‚Üí Wrapper object
        Integer objA = a;
        System.out.println("Autoboxing: Integer object value = " + objA);

        // üëá Wrapper object
        Integer objB = Integer.valueOf(50);

        // ‚úÖ Auto-unboxing: Wrapper object ‚Üí primitive
        int b = objB;
        System.out.println("Auto-unboxing: Primitive int value = " + b);

        // üî∏ Autoboxing inside collections
        java.util.ArrayList<Integer> list = new java.util.ArrayList<>();
        list.add(10); // primitive int is automatically autoboxed to Integer object
        list.add(20);
        list.add(a);  // same here

        System.out.println("ArrayList after Autoboxing: " + list);

        // üî∏ Auto-unboxing during arithmetic operations
        Integer x = 5;
        Integer y = 10;

        int sum = x + y;  // both x and y are auto-unboxed to int during addition
        System.out.println("Sum after Auto-unboxing: " + sum);
    }
}
```

### üìù **Output:**

```
Autoboxing: Integer object value = 100
Auto-unboxing: Primitive int value = 50
ArrayList after Autoboxing: [10, 20, 100]
Sum after Auto-unboxing: 15
```

## üìù **4. Key Points to Remember**

| Autoboxing                         | Auto-unboxing                               |
| ---------------------------------- | ------------------------------------------- |
| Converts **primitive ‚Üí object**    | Converts **object ‚Üí primitive**             |
| Uses `valueOf()` internally        | Uses `xxxValue()` internally                |
| Useful when storing in collections | Useful when doing arithmetic or assignments |
| Example: `Integer obj = 5;`        | Example: `int x = obj;`                     |

## ‚ö†Ô∏è **5. Caution**

Autoboxing can sometimes lead to **NullPointerException** if you try to unbox a `null` object:

```java
Integer obj = null;
int num = obj;   // ‚ùå Runtime error: NullPointerException
```

So always make sure the object is not null before unboxing.
